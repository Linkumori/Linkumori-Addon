## Section 6 — Remote Rules and Associated Risks

> ⚠️ **Warning:** Enabling Remote Rules exposes your device and network to risks beyond the control of Linkumori. Read this section carefully before enabling this feature.

**6.1 How Remote Rules Work**

The Remote Rules feature is disabled by default. When enabled, the Add-on downloads rule files from URLs specified by the User. Your browser connects directly to those third-party servers to retrieve such files. When Remote Rules are disabled, the Add-on operates entirely offline using only its built-in rules and no external network requests are made.

**6.2 Third-Party Data Collection**

Those third-party servers may collect standard web server information including, but not limited to:

- IP address and geolocation data;
- Browser type, version, and operating system;
- Request timestamps and access logs;
- Referring URLs and session metadata, etc.

Linkumori does not control, monitor, or store any data transmitted to or collected by such third-party servers.

**6.3 Disclaimer of Responsibility for Third-Party Servers**

Linkumori is not responsible for the privacy practices, data handling policies, security measures, or conduct of any third-party server from which Remote Rules are fetched. The User is strongly advised to review the privacy policy and terms of any such third-party server before enabling this feature.

**6.4 Intended Users**

This feature is intended **exclusively for advanced users** who have full understanding of the associated risks and who maintain complete control over both their physical and virtual infrastructure — including, without limitation, servers that host rules and software that manages server operations. This feature is **not recommended** for general or non-technical users.

**6.5 User as Data Controller**

Since this feature is intended exclusively for advanced users who maintain full control over both their physical and virtual infrastructure, the User is the **Data Fiduciary or Data Controller** for any data processing that occurs through that connection. The User determines the purpose (why they choose to use this feature and what they intend to achieve by connecting to their infrastructure) and the means (how data is collected, handled, and processed within their infrastructure).

Because the User controls both the physical layer (hardware, servers, and network equipment) and the virtual layer (software, operating systems, services, and configurations), every decision about that connection is the User's alone. The User may choose to log connections, restrict them, disable them entirely, or configure them in any way they see fit. Linkumori is simply the instrument through which the User's instructions are carried out; responsibility for lawful use rests entirely with the User.

**6.6 User Responsibility When Using Remote Rules**

Linkumori is a tool. When the User enables the Remote Rules feature and provides a custom URL, the Add-on simply executes a fetch request as directed by the User. Linkumori does not determine, influence, or control what URL the User adds, what server the User points to, or what data is served. Those decisions are entirely the User's.

By providing a URL, the User accepts that:

- It is the **User's responsibility** to ensure the URL provided complies with all applicable laws in their jurisdiction, including but not limited to data protection and privacy laws.
- It is the **User's responsibility** to ensure they have lawful authority over the physical and virtual infrastructure behind that URL.
- Linkumori bears no responsibility for the content, legality, or consequences of URLs the User chooses to add.

**6.7 Risks of Using Remote Rules**

By enabling Remote Rules, the User acknowledges and accepts the following risks:

- **Malicious Rules** — Rule files downloaded from untrusted or compromised servers may contain harmful, deceptive, or unintended instructions that alter Add-on behaviour.
- **Data Exposure** — Connecting to external servers may expose your IP address, network identity, and device metadata to unknown third parties.
- **Man-in-the-Middle Attacks** — Unencrypted or poorly secured connections to remote servers may be intercepted, resulting in tampered rule files being delivered to your device.
- **Server Compromise** — Even trusted servers may be compromised at any time without notice, causing previously safe rule sources to become dangerous.
- **Loss of Control** — Remote Rules are not verified or audited by Linkumori; the User assumes full responsibility for any consequences arising from their use.
- **Legal and Compliance Risk** — Rules fetched from foreign servers may introduce functionality that violates applicable Indian or local laws, for which the User bears sole responsibility, etc.

**6.8 Limitation of Liability for Remote Rules**

The User enables and uses Remote Rules entirely at their own risk. Linkumori shall bear no liability for any damage, data loss, security breach, legal consequence, or any other harm arising directly or indirectly from the use of the Remote Rules feature.

## Section 7 — Issue Reporting Feature

7.1 The issue reporting feature is an optional, user-initiated tool. Linkumori does not collect, transmit, or store any data through this feature.

7.2 When the report function is used, the Add-on constructs a GitHub issue URL or GitHub search URL entirely within the User's browser. The Add-on does not transmit any data and does not contact any server while constructing this URL. No network request is made until the User explicitly chooses to proceed.

7.3 Before any redirect takes place, the Add-on presents a consent notice, a full preview of the constructed URL in both encoded and human-readable form, and gives the User the option to copy the URL, cancel entirely, or proceed. No redirect occurs unless the User explicitly confirms.

7.4 If the User chooses to proceed to GitHub, their interaction with GitHub is governed solely by GitHub's own Privacy Statement and Terms of Service. GitHub may log data such as IP address, browser type, and URL parameters in accordance with their own policies, regardless of whether an issue is ultimately submitted.

7.5 To the extent GitHub collects, stores, uses, or otherwise processes personal data, GitHub acts as an independent Data Fiduciary or Data Controller for that processing. GitHub does not process that data on behalf of Linkumori, and Linkumori does not determine GitHub's purposes or means of processing. Linkumori has no access to, and bears no responsibility for, any data processed by GitHub once the User's browser navigates to their site.

7.6 If the User submits an issue, any information provided — including additional comments — will be publicly visible on the repository, as it is a public GitHub repository.
